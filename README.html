Configuration

When kexec-loader starts it searches for a file named kexec-loader.conf in the root directory of the following filesystems, in order:

  1. /dev/fd0
  2. /dev/fd1
  3. /dev/sda
  4. /dev/sdb
  5. /dev/sdc
  6. /dev/sdd

The configuration file should contain directives, optionally followed by some whitespace and then a value, lines beginning with the hash (#) character are ignored, as are empty lines.

  • timeout <integer>
    Sets the timeout until the default target is loaded in seconds, if no target is set as default the first target is used.
  • title <text>
    Marks the beginning of a target definition.
  • kernel <filename>
    Filename of the kernel to load.
  • append <text>
    Arguments to append to the kernel command-line.
  • initrd <filename>
    Filename of initrd image to load.
  • default
    Marks target as default to boot when timeout expires.
  • rootfs [<fstype>:]<device>
    Specifies the root filesystem, equivilent to mount with a mount-point of "/"
  • mount [<fstype>:]<device> <mount-point>
    Mounts the specified device at the specified mount-point, if fstype is ommited or set to "auto" kexec-loader will attempt to guess the filesystem type, currently detection is supported for ext2, ext3, XFS, reiserfs, minix, FAT and NTFS.
